# AI-Assisted Histopathology Cancer Detection Platform Backend

This is the production-style FastAPI backend for the **AI-Assisted Histopathology Cancer Detection Platform**. The system provides clinical staff with deep-learning-assisted classification (Benign vs. Malignant) of histopathology tissue slides using a **ResNet50** architecture, backed by **Grad-CAM region activation overlays** to resolve black-box AI explainability concerns.

All scans, patient records, and generated diagnostic summaries are saved and managed via **PostgreSQL** with an **Alembic** migration pipeline.

---

## ⚠️ Clinical Disclaimer

> [!WARNING]
> **AI-assisted screening support only. Not a medical diagnosis.**
> This platform is an assistive tool designed to guide qualified clinical pathologists. All classifications, heatmap overlays, and generated reports must be reviewed, verified, and signed off by a licensed medical professional before making patient care decisions.

## 🚀 Installation & Setup

### 1. Prerequisites
Ensure you have the following installed:
* Python 3.11+
* PostgreSQL

### 2. Configure Environment
Copy the `.env.example` template into a `.env` file inside the `backend/` directory:
```bash
cp .env.example .env
```
Adjust the `DATABASE_URL` parameter to point to your local PostgreSQL instance:
```env
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/cancer_db
```

### 3. Install Dependencies
```bash
pip install -r requirements.txt
```

### 4. Run Database Migrations
Generate database tables by running the Alembic migration command:
```bash
alembic revision --autogenerate -m "Initial schema"
alembic upgrade head
```

## 🎯 API Endpoint Specs

### 🔑 Security & Auth
* **`POST /auth/register`**: Registers a new clinician.
* **`POST /auth/login`**: Authenticates and yields a JWT access token.

### 👥 Patient Profiles
* **`POST /patients`**: Registers a new patient profile.
* **`GET /patients`**: Lists all patient profiles.
* **`GET /patients/{id}`**: Retrieves a specific patient.

### 🔬 AI Screening
* **`POST /predict`**: Uploads a histopathology biopsy slide (multipart file upload) along with `patient_id` (form parameter). Performs image preprocessing, runs the ResNet50 classifier, compiles a Grad-CAM heatmap overlay, writes a diagnostics text report, and registers the transaction.
  * **Returns**:
    ```json
    {
      "scan_id": 15,
      "prediction": "Malignant",
      "confidence": 0.92,
      "heatmap_url": "heatmaps/heatmap_xxx.png"
    }
    ```

### 📊 Reports & Logs
* **`GET /scans`**: Lists all scan history logs.
* **`GET /scans/{id}`**: Retrieves metadata and image paths for a specific scan.
* **`GET /reports/{id}`**: Retrieves the structured text diagnostics summary (with disclaimers) by Report ID.

## 🧠 AI Model Pipeline & Grad-CAM

1. **Preprocessing (`ai/preprocess.py`)**: Checks file formats, resizes images to $224 \times 224$ pixels, converts to tensors, and normalizes using standard ImageNet values (mean: `[0.485, 0.456, 0.406]`, std: `[0.229, 0.224, 0.225]`).
2. **Inference (`ai/inference.py`)**: Runs inputs through ResNet50 in evaluation mode under cached global memory contexts, returning labels and confidence percentages.
3. **Grad-CAM (`ai/gradcam.py`)**: Uses backward hooks on the last convolutional layer (`model.layer4[-1]`) to resolve classification features. Computes raw weight activation maps, overlays color-coded JET masks (Red for tumor indicators, Blue for benign backgrounds) onto original biopsy images, and saves overlays under `heatmaps/`.
4. **Training (`ai/train.py`)**: Standalone training pipeline. Run it locally to train the ResNet50 model:
   ```bash
   python app/ai/train.py --dummy --epochs 3
   ```
